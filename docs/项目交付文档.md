## 1. 项目概述

### 1.1 需求背景

实现一个纸牌匹配消除游戏，包含以下核心功能：


| 编号 | 需求           | 描述                                                                                      |
| ---- | -------------- | ----------------------------------------------------------------------------------------- |
| R1   | 手牌区翻牌替换 | 点击手牌区任意卡牌，牌 MoveTo 平移至顶部位置，成为新的匹配基准牌                          |
| R2   | 桌面牌匹配消除 | 点击桌面翻开牌，与手牌顶部牌点数差 1 则匹配；匹配后牌 MoveTo 飞入手牌区，替换为新的顶部牌 |
| R3   | 回退功能       | 支持连续回退；每次回退牌 MoveTo 反向平移回原位；直到无回退记录                            |

### 1.2 界面布局

```
┌───────────────────────────────────┐
│         主牌区  1080 × 1500        │  ← 5列拱门堆叠，顶层翻开可点击
│                                    │
│    ♠K   ♥3   ♦7    ♣A   ♠5        │    翻开 = 可点击
│    ██   ██   ██    ██   ██        │    覆盖 = 蓝色面朝下
│         ██   ██    ██             │
│              ██                   │
├───────────────────────────────────┤
│         手牌区  1080 × 580         │
│                                    │
│  [牌堆-12]   ♥A ♣J ♦5 ♣4 [♠8]    │  ← [♠8] 为顶部牌(高亮)
│                                    │
│       << 翻牌 >>       << 回退 >>   │
└───────────────────────────────────┘
```

## 2. 技术架构

### 2.1 分层架构

```
┌─────────────────────────────────────────────────┐
│                    View Layer                     │
│  GameScene                                        │
│  ├── TableAreaView   (主牌区布局 1080×1500)         │
│  ├── HandAreaView    (手牌区布局 1080×580)          │
│  └── CardSprite      (卡牌渲染 182×282)            │
│                                                   │
│  依赖: Controller (回调) + Model (const 只读)       │
├─────────────────────────────────────────────────┤
│                 Controller Layer                  │
│  GameController     (游戏逻辑编排器 - Facade)       │
│  ├── MatchRule       (匹配策略 - Strategy)          │
│  ├── UndoManager     (回退管理 - Command Stack)     │
│  └── Actions         (具体操作命令 - Command)       │
│                                                   │
│  依赖: Model 层                                     │
├─────────────────────────────────────────────────┤
│                   Model Layer                     │
│  CardTypes.h       (基础类型定义)                   │
│  GameBoard.h/cpp   (牌局状态 - 列式存储)             │
│  IAction.h         (命令抽象接口)                   │
│                                                   │
│  依赖: 无 (零外部依赖)                               │
└─────────────────────────────────────────────────┘
```

**依赖方向严格单向：Model ← Controller ← View。** 任一层的修改不影响其下层。

### 2.2 设计模式应用


| 设计模式     | 应用位置                  | 解决的问题                                                    |
| ------------ | ------------------------- | ------------------------------------------------------------- |
| **MVC**      | 全项目三层目录            | 视图、逻辑、数据独立开发和测试                                |
| **Command**  | `IAction` + `UndoManager` | 每步用户操作可撤销，回退逻辑与正向逻辑解耦                    |
| **Strategy** | `IMatchRule` 接口         | 匹配规则可插拔替换，新增规则不影响已有代码                    |
| **Observer** | Controller 回调函数       | View 订阅 Controller 状态变化，避免 Controller 持有 View 引用 |

## 3. 模块设计

### 3.1 Model 层

#### CardTypes — 基础类型


```cpp
enum class Suit { Spade, Heart, Club, Diamond };
using Rank = int;  // A=1, 2~10, J=11, Q=12, K=13

struct CardData {
    int  cardId;     // 全局唯一标识
    Suit suit;       // 花色
    Rank rank;       // 点数
    bool faceUp;     // 状态
};
```

#### GameBoard — 牌局状态

以 `vector<deque<CardData>>` 列式存储主牌区数据：


| 存储结构                   | 用途                                                   |
| -------------------------- | ------------------------------------------------------ |
| `tableColumns[col][depth]` | 主牌区第 col 列的第 depth 张牌 (front=底部, back=顶部) |
| `handCards[index]`         | 手牌区第 index 张 (back=顶部牌)                        |
| `reserveCount`             | 备用牌堆剩余张数                                       |

关键方法：`PopTopCard / PushCard / SetTopCardIndex / FindCard`

#### IAction — 命令接口

```cpp
class IAction {
public:
    virtual bool Execute() = 0;      // 正向执行
    virtual void Undo()     = 0;      // 逆向撤销
    virtual int  GetMovedCardId() const = 0;   // 被移动的牌ID
    virtual int  GetSourceCol()   const = 0;   // 原始列/位置
    virtual int  GetSourceDepth() const = 0;   // 原始深度
    virtual bool IsTableSource()  const = 0;   // 是否来自桌面
};
```

### 3.2 Controller 层

#### GameController — 逻辑编排器

对外暴露游戏操作接口，通过回调通知 View 更新：


| 方法                         | 说明                                           |
| ---------------------------- | ---------------------------------------------- |
| `OnTableCardClicked(cardId)` | 桌面牌点击 → 匹配判断 → 执行命令 → 播放动画 |
| `OnHandCardClicked(cardId)`  | 手牌点击 → 切换顶部牌 → 播放动画             |
| `Undo()`                     | 回退 → 获取命令信息 → 播放反向动画 → 撤销   |
| `DrawFromReserve()`          | 翻牌 → 抽牌加入手牌 → 播放动画               |

回调机制（解耦 Controller 与 View）：


| 回调类型           | 触发时机   | View 层处理             |
| ------------------ | ---------- | ----------------------- |
| `BoardRefreshCB`   | 数据变更后 | 刷新主牌区和手牌区渲染  |
| `CardMovedCB`      | 桌面牌匹配 | 播放匹配特效 + 移动动画 |
| `HandSwitchAnimCB` | 手牌切换   | 播放牌滑到顶部动画      |
| `UndoAnimCB`       | 回退操作   | 播放反向飞回动画        |
| `DrawAnimCB`       | 翻牌操作   | 播放旋转飞入动画        |
| `HistoryCB`        | 回退栈变化 | 更新回退按钮启用/禁用   |

#### MatchRule — 匹配策略

```cpp
class IMatchRule {
public:
    virtual bool CanMatch(const CardData& a, const CardData& b) const = 0;
};

// 默认实现：点数差1，无花色限制
class AdjacentRankRule : public IMatchRule {
    bool CanMatch(const CardData& a, const CardData& b) const override {
        return abs(a.rank - b.rank) == 1;
    }
};
```

#### UndoManager — 回退管理器

LIFO 栈管理，存储 `IAction*` 指针序列：


| 方法              | 说明                   |
| ----------------- | ---------------------- |
| `Execute(action)` | 执行命令并入栈         |
| `Undo()`          | 出栈顶命令并撤销       |
| `PeekTop()`       | 查看栈顶命令（不出栈） |
| `CanUndo()`       | 栈是否非空             |

#### Actions — 具体命令实现


| 命令类              | 触发操作   | Execute                              | Undo                                             |
| ------------------- | ---------- | ------------------------------------ | ------------------------------------------------ |
| `TableToHandAction` | 桌面牌匹配 | 牌从桌面移除→加入手牌区→翻面下方牌 | 牌从手牌区移除→放回桌面原位→恢复覆盖状态       |
| `HandSwitchAction`  | 手牌区切换 | 将选中牌移到顶部（SetTopCardIndex）  | 通过 cardId 找牌→移回原始位置（InsertHandCard） |
| `DrawReserveAction` | 从牌堆翻牌 | 无额外操作（牌已在 Execute 前加入）  | 从手牌区移除牌→恢复牌堆计数                     |

### 3.3 View 层

#### GameScene — 主场景

- 1080×2080 设计分辨率
- 场景级触摸事件处理（命中检测按 z-order 排序）
- 动画编排（匹配特效、MoveTo、翻牌旋转、回退飞回）
- 动画锁 `m_animating` 防止操作打断

#### CardSprite — 卡牌渲染

```cpp
static CardSprite* CreateWithCard(const CardData& card, bool useBig);
void UpdateCard(const CardData& card);     // 刷新渲染
void MoveToPosition(Vec2 target, float dur, callback);  // 移动动画
void PlayMatchEffect(callback);            // 匹配弹跳特效
void PlayFlipAnim(callback);               // 翻转动画
bool HitTest(const Vec2& worldPos);        // 命中检测
```

渲染：`card_general.png`(背景) + 数字PNG + 花色PNG 三图合成。红色花色(Heart/Diamond)自动选红色数字，黑色套装(Spade/Club)选黑色数字。

#### TableAreaView — 主牌区布局

- 区域：1080×1500
- 拱门结构：5列，高度分布 {2,3,4,3,2}
- 列内堆叠：每张牌向上偏移 50px
- 拱门效果：中间列顶部牌比边缘列高 40-80px

#### HandAreaView — 手牌区布局

- 区域：1080×580
- 平滑过渡：牌少（≤5）完全平铺，牌多逐渐重叠
- 最小可见宽度：55px/张（确保数字和花色可见）
- 翻牌堆：左侧斜向扇形堆叠，显示剩余张数

## 4. 核心流程

### 4.1 桌面牌匹配流程

```
用户点击桌面翻开牌
    │
    ▼
GameScene::onTouchBegan
    │  遍历所有卡牌，按 z-order 从高到低命中检测
    ▼
GameController::OnTableCardClicked(cardId)
    │
    ├─ FindCard(cardId)          ← 验证卡牌存在
    ├─ IsCardTopOfColumn()       ← 验证是顶部牌
    ├─ MatchRule::CanMatch()     ← 点数差1判定
    │
    ▼
new TableToHandAction → UndoManager::Execute()
    │
    ├─ Execute():
    │    ├─ PopTopCard()          牌从桌面移除
    │    ├─ SetCardFaceUp()       下方牌翻面
    │    └─ PushHandCard()        加入手牌区
    │
    ▼
CardMovedCB → OnCardMoved()
    │
    ├─ PlayMatchEffect()         弹跳特效 (0.16s)
    └─ MoveToPosition()          平移到手牌区顶部 (0.2s)
            │
            ▼
        OnBoardChanged()
            ├─ TableAreaView::RefreshFromBoard()
            ├─ HandAreaView::RefreshFromBoard()
            └─ UndoButton 状态更新
```

### 4.2 回退流程

```
用户点击 "<< 回退"
    │
    ▼
GameController::Undo()
    │
    ├─ CanUndo()                 检查是否有历史
    ├─ PeekTop()                 获取栈顶命令信息
    │   ├─ GetMovedCardId()      获取卡牌ID
    │   ├─ GetSourceCol/Depth()  获取原位
    │   └─ IsTableSource()       判断目标区域
    │
    ▼
UndoAnimCB → OnUndoAnim()
    │
    ├─ 计算目标位置              桌面/手牌原位
    └─ MoveToPosition()          反向飞回 (0.2s)
            │
            ▼
        UndoManager::Undo()
            ├─ 出栈命令
            └─ 撤销数据 (牌移回原位)
            │
            ▼
        OnBoardChanged()
            └─ 全量 View 刷新
```

### 4.3 状态管理

- `m_animating` 动画锁：任何动画进行中阻止新的操作，防止并发导致数据错乱
- 动画完成后统一调用 `OnBoardChanged()` 刷新，不在动画中途修改 View

    int coins    = 100;   // 金币

int wildCardCost = 50;  // 使用一次万能卡消耗 50 金币

#### 业务逻辑

```cpp
// Controller/GameController.cpp
bool GameController::UseWildCard() {
    if (m_playerData.coins < m_board.wildCardCost) return false;   // 余额不足
    if (m_board.ReserveCount() <= 0) return false;                 // 无牌可换

    m_playerData.coins -= m_board.wildCardCost;                    // 扣金币
    CardData wild = m_board.DrawFromReserve();
    wild.cardType = CardType::Wild;
    wild.faceUp   = true;
    m_board.PushHandCard(wild);

    m_undoManager.Execute(new UseItemAction(&m_board, &m_playerData,
        wild.cardId, m_board.wildCardCost));
    NotifyBoardRefresh();
    return true;
}

// 回退时退回金币
void UseItemAction::Undo() {
    m_player->coins += m_cost;              // 退回金币
    m_board->PopHandCard(m_handIndex);      // 移除万能牌
    m_board->SetReserveCount(m_board->ReserveCount() + 1);  // 归还牌堆
}
```

| 新动画效果  | V         | 1               | ~20      |
| 新牌区/布局 | M + C + V | 4               | ~150     |

核心设计原则：**采用开放封闭原则，新增功能通过扩展接口实现，不修改现有代码。** Command 模式使回退系统对新增操作透明；Strategy 模式使匹配规则独立于业务流程。
